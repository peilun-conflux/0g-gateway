# 存储网关 对接说明

本网关提供**标准 S3 兼容**的对象存储能力,可替换华为 OBS。后端用**任意 S3 SDK 或华为 OBS SDK**把 endpoint 指向本网关即可,沿用原来的桶名 + 对象 key 读写,改动很小。

## 一、如何连接

- **服务地址(endpoint)**:`http://<网关地址>:8080`
- **协议**:S3,**路径风格(path-style)**
- **AK/SK**:任意非空值即可(当前不验签名,见第三节)

华为 OBS **Java** SDK（`esdk-obs-java-bundle`，已实测打通）这样配置:

```java
ObsConfiguration config = new ObsConfiguration();
config.setEndPoint("<网关地址>");        // 仅主机名/IP
config.setEndpointHttpPort(8080);
config.setHttpsOnly(false);              // 内网 http
config.setPathStyle(true);               // 必须:路径风格
config.setAuthTypeNegotiation(false);
config.setAuthType(AuthTypeEnum.V2);     // 必须:S3 v2 签名
ObsClient client = new ObsClient("anyAK", "anySK", config); // AK/SK 任意非空
```

> **无需** `setVerifyResponseContentType(false)` —— 网关已在服务端把 XML 响应规范化,SDK 默认严格校验即可用。
> Node.js SDK 同理(`signature:'v2'` + `path_style:true` + 完整 URL `server`);标准 AWS SDK / boto3 强制 path-style 即可。

## 二、支持的操作

| 操作 | 说明 |
|---|---|
| 上传对象 PutObject | 请求体为文件内容;桶 + key 沿用原 OBS。支持条件写(If-Match / If-None-Match) |
| 下载对象 GetObject | 支持 Range(断点续传 / 分片下载) |
| 取对象头 HeadObject | 仅取元数据(大小、Content-Type、ETag) |
| 删除对象 DeleteObject / 批量删 | 删不存在的 key 不报错(S3 语义) |
| 覆盖写 | 重新 PUT 同 key 即覆盖 |
| 复制对象 CopyObject | 零拷贝(同内容不重传);经华为 OBS SDK 实测可用 |
| 列举 ListObjects | 支持 prefix 前缀过滤(**无分页**) |
| 桶操作 | 建桶 / 列桶 / 探测桶 / 删空桶;向不存在的桶写会**自动建桶** |
| 图片缩放预览 | `GET .../<bucket>/<key>?x-image-process=image/resize,w_200,h_100,m_lfit`,实时缩略图(仅 resize) |

对象的文件名、Content-Type 会作为元数据保存,下载时原样返回;ETag = 内容 MD5。

> 对接示例:`putObject(bucket, "user/123/avatar.png", ...)` 上传,`getObject(...)` 下载——和 OBS 用法基本一致。

## 三、与华为 OBS 的区别

| # | 区别 | 说明 |
|---|---|---|
| 1 | **不验证签名** | 当前不校验 S3 签名,AK/SK 任意值都通过。写安全靠**只绑内网**,务必绑内网地址勿暴露公网。「服务端 AK/SK 鉴权」待后续实现。 |
| 2 | **无分段上传** | 不支持 Multipart Upload(库层用内存攒分片,仅适合小文件),单对象上限 4 GB。 |
| 3 | **不支持空文件** | 0 字节对象会被拒绝(返回 400)。 |
| 4 | **List 无分页** | 列举支持 prefix 过滤,但不支持分页;无版本控制(Versioning)。 |
| 5 | **异步上链** | PUT 返回即可读(已落本地缓存),真正写入 0G 由后台异步完成;勿假设 PUT 刚返回就已永久落链。 |
| 6 | **删除=移除键映射** | 删除后对象在桶内不可访问;0G 数据内容寻址、不可物理擦除,有合规擦除要求需单独讨论。 |

最常用的上传 / 下载 / 删除 / 覆盖 / 列举均已支持。详细接口文档见 `migration-from-obs.md`。
